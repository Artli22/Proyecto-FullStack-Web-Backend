# Bienvenido al repositorio Backend del proyecto Full Stack
## Aqui se guardaron aquellos archivos que consistuyen a la logica de negocios con la base de datos series2.db

### Tecnologias Implementadas
- Go
- SQLite
- Versel para deploytment
- Turso para envirotmental keys
### Lista de challenges implementados
- Calidad visual del cliente (Yo digo que si)
- Calidad del historial de Git (Yo digo que si)
- Organización del código (Yo digo que si)
- Códigos HTTP correctos en toda la API 
- Validación server-side con respuestas de error en JSON descriptivas
- Paginación en GET /series con parámetros ?page= y ?limit=
- Búsqueda por nombre con ?q=
- Ordenamiento con ?sort= y ?order=asc|desc

### Aprendizaje de tecnologias utilizadas
#### Sinceramente me arrepiento de haber escogido SQLite como la tecnologia para la base de datos porque complico mucho el deployment del trabajo, sin emabrgo, esta decision me llevo a conocer herramientas de alto potencial para los deployments de los repositorios en github, tales como: Versel, Turso y Railway. Me gusto mucho las tres herramientas previamente mencionadas porque tenian interfaces intuitivas, logs extremadamente detallados al momento de ocurrir un error en el deploy y reacciones casi instantaneas al momento de hacer un nuevo deployment.
